All About DBC's Pair Programming

What is pair programming?

Pair programming is a technique in which two programmers pair together on one work station. One programmer will be in the driver role while the other programmer will be in the navigator role. They work together frequently switching to reduce errors, collaborate on problem-solving/trouble-shooting, and expressing ideas from code to English.

Why is pair programming important to DBC?

Pair programming is important to DBC because it creates clear communication between team members. People misconstrue that programming is a single body activity whereas it actually requires a collaborative approach. It helps reduce coding errors, increases the quality of code, and encourages programmers to share knowledge with each other.

Compare Driver and Navigator

Common misconceptions about Driver and Navigator roles.

  1. People reverse the roles.
  2. Drivers are the ones who are dictating the code.
    • Mythbusted: Drivers write the code!
  3. Navigators are the ones who type the code.
    • Mythbusted: Navigators dictate what the code should be, navigating the driver.
  4. Driver/Navigator roles are rigid.
    • Mythbusted: The roles are actually fluid and change frequently.
    • Both parties discuss adjustments to the code and content. No one is the dictator here.
  5. Both parties have to be the same skill level.
    • Mythbusted: The pair can be any skill level, which offers an opportunity to learn from each other, both as a novice and as an expert.

Feedback

Feedback should try be to A.S.K:

  • Actionable: The feedback you give should be able to be implemented.
  • Specific: Give feedback that targets an area that can be improved or highlights something that was well done during the session.
  • Kindness: Honesty is a main part of kindness. Feedback should be truthful and constructive.
